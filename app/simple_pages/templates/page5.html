Glossary of OOP terms

OOP or Object Oritented Programming is a model that uses objects and classes in programming. It is considered the standard way to code for programmers as the structure uses classes (the blueprint) to create different instances of objects. Classes show the overall broader category, and they have individual smaller objects. For example, a food class can have different types of cuisines as the properties and then an instance of the food to represent different types of specific foods in the category.

Instantiation

Instantiation is the realization of a predefined object. In OOP (object-oriented programming), a class of object may be defined. An instance of that object may then be declared, giving it a unique, named identity so that it may be used in the program. This process is called "instantiation". It basically creates an object/instance for a class which inherits all class variables and methods. In the calculator example, it would be the calculator = Calculator() that instantiates the calculator with all of its properties from the Calculator. Another example is the Addition.create(value) – instantiates and add it to the create, they all call add calculations method to add all the calculations.

Object

An object in OOPS is nothing but a self-contained component which consists of methods and properties to make a particular type of data useful. For example color name, table, bag, barking. When you send a message to an object, you are asking the object to invoke or execute one of its methods as defined in the class. Individual objects represent specific examples of the abstract class like the food example, type of cuisine, then specific types of food from that category is known as objects. An example from the calculator project would be, Def add(tuple_list) where the function is to create an addition object, result to get the result of the addition object.

Class

A class is a blueprint for how something should be defined. It doesn’t actually contain any data. Classes are used to create user-defined data structures. Classes define functions called methods, which identify the behaviors and actions that an object created from the class can perform with its data. In the calculator project, it has a calculator class that has methods to add, subtract, and get result. This is the action that an object can do.

Namespace

A namespace is a collection of names and the details of the objects referenced by the names. We can consider a namespace as a python dictionary which maps object names to objects. The keys of the dictionary correspond to the names and the values correspond to the objects in python. This makes it easy to identify them as there can be a lot of objects. The types of namespaces are built-in namespaces, global namespaces, local namespaces, and enclosing namespaces.

Constructor

The constructor is a method that is called when an object is created. This method is defined in the class and can be used to initialize basic variables. Every class has a constructor, but its not required to explicitly define it. For the calculator project, the constructor is created with the function test_calculator_get_result_method with no parameters. They also have intialized calc1.result = 5.
