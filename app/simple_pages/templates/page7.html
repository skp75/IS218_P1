Object Oriented Programming

OP or Object Oritented Programming is a model that uses objects and classes in programming. It is considered the standard way to code for programmers as the structure uses classes (the blueprint) to create different instances of objects. The Calculator project was a template made for the use

Encapsulation

Encapsulation describes the concept of bundling data and methods within a single unit. So, for example, when you create a class, it means you are implementing encapsulation. A class is an example of encapsulation as it binds all the data members (instance variables) and methods into a single unit. For example, in the calculator project, calculation = addition.create(tuple) is showcasing encapsulation because it holds its data and method in one object in one place. It combines its functionality. An example from the calculator project would be: calculation = addition.create(tuple) because it holds its data and method in one object in one place.

Inheritance

Inheritance refers to defining a new class with little or no modification to an existing class. The new class is called derived (or child) class and the one from which it inherits is called the base (or parent) class. It is like a tree where it branches from the top and everything can be accessed from the top (parent class). In the calculator project, class addition(calculation) is inheritance. In order to obey the single principle, there is one method that creates any type of calculation type class.

Polymorphism

Polymorphism is the ability of any data to be processed in more than one form. The most common use of polymorphism in object-oriented programming occurs when a parent class reference is used to refer to a child class object. Here we will see how to represent any function in many types and many forms. Also, polymorphism lets us define methods in the child class that have the same name as the methods in the parent class.

Abstraction

Abstraction is a process of hiding the real implementation of the method by only showing a method signature. An example from the calculator projetc would be: calculation = addition.create(tuple)- Abstraction because the add method calls another method that does another thing that is more complicated. This is both part of abstract and encapsulation.
