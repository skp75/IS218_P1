CI/CD

CI/CD is the combined practices of continuous integration and either continuous delivery or continuous deployment. CI/CD bridges the gaps between development and operation activities and teams by enforcing automation in building, testing and deployment of applications.

Containers

A container is a standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another. A Docker container image is a lightweight, standalone, executable package of software that includes everything needed to run an application. It is made possible by process isolation and virtualization that is built into the Linux Kernel.

Images

A Docker image is a file used to execute code in a Docker container. Docker images act as a set of instructions to build a Docker container, like a template which contains a set of instructions.

Docker Commands

  • docker run: This command is used to create a container from an image
  • docker ps: This command is used to list the running containers with their information ex. container id
  • docker ps -a: This command is used to show all the running and exited containers
  • docker stop: This command stops a running container. Information can be found from docker ps
  • docker kill: This command kills the container and stops the process immediately. This forcefully kills is and overwrites any error
  • docker commit: This command creates a new image of an edited container on the local system
  • docker push: This command is used to push an image to the docker hub repository
  • docker pull: This command is used to pull images from the docker repository
  • docker rm: This command is used to delete a stopped container
  • docker build: This command is used to build an image from a specified docker file

How to start the project and how to access the container with the terminal using just terminal commands and not PyCharm

To start a project and access it just with the terminal, you need to open up a new terminal. This can be git bash or powershell. Firstly, open up a new project and on the terminal you need to use the build command to build a container image that you can see in the docker hub. This shows if the container is active and the port that is assigned to it. You can also view the container in the terminal using docker ps which shows all the 'online' containers and their value ids.


You can also run the container as any time to make it active by using the docker run -dp command followed by the container name. This will create a mapping of the host port too. All this can once again be seen on the docker hub menu too.
