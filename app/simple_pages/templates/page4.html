CI/CD

CI/CD is the combined practices of continuous integration and either continuous delivery or continuous deployment. CI/CD bridges the gaps between development and operation activities and teams by enforcing automation in building, testing and deployment of applications. CI or "Continuous Integration is a software development practice where members of a team integrate their work frequently, usually each person integrates at least daily - leading to multiple integrations per day. Each integration is verified by an automated build (including test) to detect integration errors as quickly as possible" (Fowler, 2006).


CD or "Continuous Delivery is a software development discipline where you build software in such a way that the software can be released to production at any time... You achieve continuous delivery by continuously integrating the software done by the development team, building executables, and running automated tests on those executables to detect problems. Furthermore, you push the executables into increasingly production-like environments to ensure the software will work in production" (Fowler, 2006)


How the Project was set up to use Github and Github actions

The Project was set up to use Github in two different ways. Firstly, on Pycharm it was set up that whenever you commit the project, it updates the changed files on Github. This not only keeps a source online, but makes sure you can go to a previous commit if need be. Secondly, another thing that was used is the Github Actions. This is where secrets were added for the docker username (DOCKER_USERNAME), docker password (DOCKER_PASSWORD), and the heroku api key from the app (HEORUKU_API_KEY). This allowed Github to connect to the docker container and Heroku app and update it or deploy it over there.


How to deploy master to a production server

To deploy a master to a production server you need to create a production file. In this project there is the prod.yml and prod_awake.yml that when you send a pull request to the master, it is able to deploy the same last changes you made to the developmental stage to the prod app and deploy it.


How to deploy an image of your project to Dockerhub when master is updated

To deploy an image of the project to the docker hub when the master is updated, it creates a container on Dockerhub automatically because of the Docker file in the project. This is also from the secret key that was entered at the start that allows this access.
