SOLID (Five Principles Of OOP)

Git is software tracking changes in any set of files, usually for coordinating work among programmers developing source code during software development. It is a DevOps tool for source code management. It is free and open-source that enables multiple developers working together

Branches

A branch is a version of the repository that diverges from the main working project which is the master branch It allows work to continue without changing the master branch. A Git project can have more than one branch. These branches are a pointer to a snapshot of your changes. When you want to add a new feature or fix a bug, you spawn a new branch to summarize your changes.

Merge

Merging is a process to connect the forked history. It is able to bring the contents of another branch to another repo. It joins two or more development history together. Merge all commits on the specified branch not already present on the current branch. It is used to maintain distinct lines of development; at some stage, you want to merge the changes in one branch.

Commit

Commits records the changes you make in a repository. It has data and a commit message at each state that tells you the new update you made. It then fetches updates from the stage environment. You can revert back to an old commit incase you have made a mistake or need to see an older version.

Tags

Tags make a point as a specific point in Git history. They can be used to mark a commit stage if it is relevant and you need to come back to it. Or the version of something. They also do not change once initiated and can be used for teh Master or developing stage.

Repositories

A repository stores all your files for a project. It also includes the history and commits you have made thus far. They can be public or private on git depending on your prefernce
